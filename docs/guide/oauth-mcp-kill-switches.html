OAuth and MCP Kill Switches ​

OpenWave OAuth and Remote MCP are controlled by OpenWave Identity. Operators can turn access off globally, per environment, per owner type, per client, or per user grant.

Global switches ​

SwitchDefaultEffect
oauth.globaloffBlocks token issuance and token introspection.
mcp.globaloffBlocks Remote MCP tool execution.
mcp.mutationsoffBlocks Remote MCP write tools even when a token has write scope.
environment.sandboxonAllows sandbox OAuth access.
environment.liveoffAllows live OAuth access.
owner.NEPTUNEonAllows Neptune owner clients.
owner.MERCHANToffAllows merchant clients.
owner.BANKoffAllows bank clients.
owner.CUSTOMERoffAllows customer delegated clients.

Emergency actions ​

Use the narrowest control that contains the issue:

  1. Disable a single client when one integration is compromised.
  2. Revoke a user’s grants when delegated access is wrong.
  3. Disable live access when production access must pause but sandbox can continue.
  4. Disable mcp.global when agent access must stop immediately.
  5. Disable oauth.global only when all OAuth access must stop.

Existing raw merchant keys and portal sessions continue to work unless their separate controls are changed.

Audit ​

Every switch change, client suspension, secret rotation, token revocation, and user-grant revocation is written to the Identity portal audit log.

Remote MCP tool calls are also logged in Astro admin audit with the tool name and result status. Tool logs must not include API keys, OTPs, webhook secrets, raw national IDs, or full account numbers.
